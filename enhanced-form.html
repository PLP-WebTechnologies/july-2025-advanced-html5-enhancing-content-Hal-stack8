Resources & Notes

  1. Field labels are associated with inputs using for and id.
  2. Validation uses required, pattern, min/max, and type="email".
  3. Form uses autocomplete where appropriate for accessibility and UX.

Accessible features

  • Semantic tags: header, nav, main, section, aside, footer, fieldset, legend.
  • ARIA attributes used where needed (aria-describedby, aria-label).
  • Keyboard accessible controls and logical tab order.

Submission checklist

  • Well-structured content using lists and a table.
  • Complete HTML5 form with many input types.
  • Correct use of placeholder, required, autocomplete, readonly.
  • HTML5 validation features implemented.
  • Clear, accessible layout using semantic tags.
